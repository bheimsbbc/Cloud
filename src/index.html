Cloud

Begriffe:

Application Gateway:

Ein Azure Application Gateway ist ein verwalteter Dienst, der den Datenverkehr auf Anwendungsebene (Schicht 7) lastenausgleicht.

Authentication:

Authentifizierung ist der Prozess, bei dem die Identität eines Benutzers oder Systems überprüft wird, um sicherzustellen, dass sie sind, wer sie vorgeben zu sein.

Authorization:

Autorisierung ist der Prozess, bei dem festgelegt wird, welche Ressourcen oder Aktionen ein authentifizierter Benutzer oder Systemzugriff hat.

Availability Set:

Ein Verfügbarkeitssatz in Azure ist eine Gruppe von virtuellen Maschinen, die darauf ausgelegt sind, Ausfallzeiten zu minimieren, indem sie auf unterschiedliche physische Hardware innerhalb eines Rechenzentrums verteilt werden.

Availability Zone:

Eine Verfügbarkeitszone in Azure ist ein isoliertes Rechenzentrumsgebiet innerhalb einer Region, das unabhängige Infrastruktur und Stromversorgung besitzt.

Azure Advanced Threat Protection:

Azure Advanced Threat Protection (ATP) ist ein cloudbasierter Sicherheitsdienst von Microsoft, der Unternehmen dabei hilft, erweiterte Bedrohungen zu erkennen, zu analysieren und darauf zu reagieren.

Azure Advisor:

Azure Advisor ist ein Azure-Dienst von Microsoft, der Empfehlungen zur Optimierung von Kosten, Leistung, Sicherheit und Zuverlässigkeit Ihrer Azure-Ressourcen gibt.

Azure Cost Management:

Azure Cost Management ist ein Microsoft-Dienst, der Unternehmen dabei unterstützt, ihre Ausgaben in der Azure-Cloud zu überwachen, zu analysieren und zu optimieren.

Azure DDoS Protection:

Azure DDoS Protection ist ein cloudbasierter Dienst von Microsoft, der Unternehmen vor Distributed Denial of Service (DDoS)-Angriffen schützt.

Azure Firewall:

Azure Firewall ist ein cloudbasierter Dienst von Microsoft, der Unternehmen dabei hilft, den eingehenden und ausgehenden Datenverkehr in Azure zu kontrollieren und zu sichern.

Azure Free Tier:

Das Azure Free Tier ist ein Angebot von Microsoft, das es Entwicklern ermöglicht, kostenlos Azure-Ressourcen zu nutzen, um Anwendungen zu erstellen, zu testen und zu entwickeln.

Azure Functions:

Azure Functions sind serverlose Funktionen von Microsoft, die Entwicklern ermöglichen, Code in verschiedenen Programmiersprachen auszuführen, ohne sich um die Verwaltung von Infrastruktur kümmern zu müssen.

Azure Germany Services:

Azure Germany Services sind eine spezielle Version von Azure für Kunden in Deutschland, die strengere Datenschutz- und Compliance-Anforderungen haben.

Azure Government Services:

Azure Government Services sind eine spezielle Version von Azure für die US-Regierung, die spezielle Sicherheits- und Compliance-Anforderungen erfüllt.

Azure Information Protection (AIP):

Azure Information Protection ist ein cloudbasierter Dienst von Microsoft, der Unternehmen dabei hilft, sensible Daten zu klassifizieren, zu kennzeichnen und zu schützen, sowohl innerhalb als auch außerhalb der Organisation.

Azure Policies:

Azure Policies sind Verwaltungsregeln in Azure, die Unternehmen dabei unterstützen, ihre Cloudumgebung zu steuern und zu verwalten, indem sie Richtlinien zur Ressourcenkonfiguration und -verwaltung festlegen.

Azure Service Health:

Azure Service Health ist ein Dienst von Microsoft, der Kunden aktuelle und geplante Service-Statusinformationen für Azure-Dienste liefert.

Blob Storage:

Blob Storage ist ein Azure-Dienst, der es ermöglicht, unstrukturierte Daten wie Texte, Bilder oder Videos in der Cloud zu speichern und darauf zuzugreifen.

Cloud Stack:

Cloud Stack bezieht sich auf die verschiedenen Schichten oder Komponenten der Cloud-Infrastruktur, die Dienste für Bereitstellung, Verwaltung und Nutzung von Cloud-Ressourcen bieten.

Cloud-Shell:

Cloud Shell ist eine von Microsoft gehostete interaktive Shell-Umgebung, die es ermöglicht, Azure-Ressourcen direkt über den Webbrowser zu verwalten, ohne dass zusätzliche lokale Tools installiert werden müssen.

Compliance Manager:

Compliance Manager ist ein Dienst von Microsoft, der Unternehmen dabei unterstützt, ihre Compliance mit Datenschutz- und Sicherheitsstandards zu verwalten und zu dokumentieren.

Consumption-Based Model:

Das "Consumption-Based Model" bedeutet, dass Kunden nur für die tatsächlich genutzten Ressourcen bezahlen.

Container:

Container sind eine standardisierte Möglichkeit, Anwendungen zu verpacken und auszuführen, die es ermöglicht, sie unabhängig von der Umgebung konsistent bereitzustellen.

Content Delivery Network:

Ein Content Delivery Network (CDN) ist ein Netzwerk von Servern, das Inhalte effizient an Endbenutzer liefert, indem Kopien an verschiedene Standorte verteilt werden.

Defense in Depth:

Defense in Depth ist ein Sicherheitskonzept, das darauf abzielt, Systeme vor Angriffen zu schützen, indem mehrere Sicherheitsschichten implementiert werden, um potenzielle Schwachstellen abzudecken.

Economies of Scale:

Economies of Scale bezieht sich auf die Kostenersparnisse, die Unternehmen erzielen, wenn sie größere Mengen produzieren oder Dienstleistungen erbringen, da die Kosten pro Einheit tendenziell sinken, wenn die Produktion oder der Betrieb zunimmt.

Egress Traffic:

Egress Traffic bezeichnet den Datenverkehr, der aus einem Netzwerk herausgeht, z. B. von einem Rechenzentrum zu einem Endbenutzer oder zu einem anderen Netzwerk.

Elasticity:

Elasticity bedeutet, dass eine Cloud-Infrastruktur sich automatisch an Änderungen in der Nachfrage anpasst, indem sie Ressourcen entsprechend skalieren.

Enterprise Subscription:

Enterprise-Abonnement ist ein Angebot von Azure für Unternehmen, das erweiterte Funktionen und Unterstützung bietet, um die Anforderungen großer Organisationen zu erfüllen.

Fault Tolerance:

Fault Tolerance bedeutet, dass ein System Ausfälle tolerieren kann, indem redundante Komponenten vorhanden sind.

File Storage:

File Storage ist ein Azure-Dienst, der die Speicherung von unstrukturierten Daten wie Dateien in der Cloud ermöglicht.

GDPR (General Data Protection Regulation):

GDPR steht für die Datenschutz-Grundverordnung (General Data Protection Regulation) und ist eine EU-Verordnung zum Schutz der persönlichen Daten von EU-Bürgern.

General Availability:

General Availability (GA) bedeutet, dass ein Produkt oder eine Dienstleistung für die allgemeine Nutzung und den kommerziellen Betrieb freigegeben wurde und vollständig unterstützt wird.

Hybrid Cloud:

Hybrid Cloud bezieht sich auf eine IT-Infrastruktur, die aus einer Kombination von öffentlicher Cloud und privater Cloud besteht, wodurch Unternehmen die Vorteile beider Ansätze nutzen können.

IaaS (Infrastructure-as-a-Service):

IaaS steht für "Infrastructure as a Service" und bezieht sich auf einen Cloud-Computing-Dienst, bei dem Unternehmen grundlegende IT-Infrastrukturkomponenten wie virtuelle Maschinen, Speicher und Netzwerke von einem Cloud-Anbieter beziehen und nach Bedarf nutzen können.

Ingress Traffic:

Ingress Traffic bezeichnet den Datenverkehr, der in ein Netzwerk eintritt, z. B. von einem Endbenutzer oder einem anderen Netzwerk in ein Rechenzentrum.

Initiatives:

Initiatives sind eine Möglichkeit, Richtlinien und Compliance-Anforderungen in Azure Policy zu organisieren und zu verwalten. Sie bieten eine Struktur, um Gruppen von Richtlinien zu organisieren und gemeinsam auf bestimmte Compliance-Ziele hinzuarbeiten.

Key Vault:

Key Vault ist ein Azure-Dienst, der es ermöglicht, kryptografische Schlüssel, Geheimnisse und Zertifikate sicher zu speichern und zu verwalten.

Locks:

Locks sind Sicherheitsmechanismen in Azure, die verhindern, dass Ressourcen versehentlich gelöscht oder geändert werden, indem sie schreibgeschützt oder gesperrt werden.

Multifactor Authentication:

Multifactor Authentication (MFA) ist ein Sicherheitsverfahren, das den Zugriff auf ein Konto oder eine Anwendung durch die Verwendung von mehreren Identitätsnachweisen erfordert, wie z.B. Passwort, SMS-Code oder biometrische Daten, um die Sicherheit zu erhöhen.

Network Security Group:

NSG steht für "Network Security Group" und ist ein Azure-Dienst, der es ermöglicht, den ein- und ausgehenden Datenverkehr zu Azure-Ressourcen über Netzwerkregeln zu steuern und zu überwachen.

Network Watcher:

Network Watcher ist ein Azure-Dienst, der Netzwerküberwachung und -diagnose für Azure-Ressourcen bietet, um Probleme zu identifizieren, die Leistung zu optimieren und die Sicherheit zu verbessern.

NIST:

NIST steht für National Institute of Standards and Technology, eine US-Bundesbehörde, die Standards und Richtlinien für Technologie und Sicherheit entwickelt und veröffentlicht.

On-Premises:

On-Premises bedeutet, dass die IT-Ressourcen lokal in der eigenen Organisation betrieben werden, im Gegensatz zur Nutzung von Cloud-Diensten.

Operational Expenditure:

Operational Expenditure (OpEx) sind laufende betriebliche Ausgaben, die regelmäßig anfallen, um ein Unternehmen am Laufen zu halten, wie z. B. Mieten, Gehälter und Wartungskosten.

PaaS (Platform-as-a-Service):

PaaS steht für "Platform as a Service" und bezieht sich auf einen Cloud-Computing-Dienst, der eine Plattform und Umgebung für die Entwicklung, Bereitstellung und Verwaltung von Anwendungen bereitstellt, ohne dass sich Benutzer um die zugrunde liegende Infrastruktur kümmern müssen.

Privat Cloud:

Eine Private Cloud ist eine Cloud-Computing-Umgebung, die ausschließlich für eine einzelne Organisation bereitgestellt und verwaltet wird. Sie bietet ähnliche Vorteile wie öffentliche Clouds, ermöglicht aber mehr Kontrolle über Sicherheit, Datenschutz und Konfiguration.

Private Preview:

Private Preview ist eine Testphase, in der ein neues Produkt oder eine neue Dienstleistung nur für eine ausgewählte Gruppe von Benutzern zugänglich ist, um Feedback zu sammeln und Verbesserungen vorzunehmen, bevor es allgemein veröffentlicht wird.

Public Cloud:

Eine Public Cloud ist ein Cloud-Computing-Modell, bei dem Dienste wie Speicher und Rechenleistung über das Internet von Drittanbietern bereitgestellt und von mehreren Kunden gemeinsam genutzt werden.

Public Preview:

Public Preview ist eine Phase, in der ein Produkt oder eine Dienstleistung einer breiteren Öffentlichkeit zugänglich gemacht wird, um Feedback zu sammeln und letzte Anpassungen vorzunehmen, bevor es allgemein verfügbar wird.

RBAC (Role Based Access Control):

RBAC (Role-Based Access Control) steuert den Zugriff auf Ressourcen basierend auf Benutzerrollen.

Resilienz:

Resilienz bezieht sich auf die Fähigkeit eines Systems, sich nach Störungen oder Problemen zu erholen und seine Funktionalität aufrechtzuerhalten.

Resource Groups:

Resource Groups sind Container zur Organisation und Verwaltung von Cloud-Ressourcen.

Resources:

Resources sind Elemente wie virtuelle Maschinen, Datenbanken oder Netzwerke, die in der Cloud erstellt und verwendet werden, um bestimmte Aufgaben auszuführen.

SaaS (Software-as-a-Service):

SaaS (Software as a Service) ist ein Cloud-Computing-Modell, bei dem Softwareanwendungen über das Internet bereitgestellt und von einem Anbieter verwaltet werden. Benutzer können auf diese Anwendungen über Webbrowser oder APIs zugreifen, ohne dass sie die zugrunde liegende Infrastruktur verwalten müssen.

Scalability:

Skalierbarkeit bezieht sich darauf, wie gut ein System in der Lage ist, mit zunehmender Arbeitslast umzugehen, indem es Ressourcen dynamisch hinzufügt oder entfernt, um die Leistung aufrechtzuerhalten.

Serverless Computing:

Serverless Computing ermöglicht das Erstellen und Ausführen von Anwendungen, ohne sich um die Verwaltung von Servern kümmern zu müssen.

Shared Responsibility:

Geteilte Verantwortung bedeutet, dass sowohl der Cloud-Anbieter als auch der Kunde für die Sicherheit der Daten und Systeme verantwortlich sind.

SLA (Service Level Agreement):

SLA (Service Level Agreement) ist ein Vertrag zwischen einem Dienstanbieter und einem Kunden, der die vereinbarten Leistungsparameter, wie Verfügbarkeit, Reaktionszeit und Support, festlegt.

Subscription:

Eine Subscription bezieht sich auf das Abonnementmodell, bei dem Kunden regelmäßig für den Zugriff auf Dienste oder Produkte bezahlen, oft in regelmäßigen Intervallen wie monatlich oder jährlich.

Tag:

Ein Tag ist ein Schlüsselwort oder eine Bezeichnung, die verwendet wird, um Ressourcen in der Cloud zu kategorisieren oder zu kennzeichnen. Tags ermöglichen eine einfachere Verwaltung, Organisation und Abrechnung von Ressourcen.

TCO Calculator:

TCO (Total Cost of Ownership) bezieht sich auf die Gesamtkosten, die mit dem Erwerb, Betrieb und Wartung von IT-Ressourcen wie Hardware, Software und Infrastruktur verbunden sind, über ihren gesamten Lebenszyklus hinweg.

Tenant:

Ein Tenant ist eine Organisation, ein Unternehmen oder eine Einheit, die eine bestimmte Menge an Ressourcen in einer Cloud- oder SaaS-Umgebung nutzt. Ein Tenant kann eine einzelne Person, ein Unternehmen oder eine Gruppe von Benutzern sein, die gemeinsam auf die Ressourcen zugreifen.

Trust Center:

Ein Trust Center bietet Informationen und Richtlinien zur Sicherheit und Compliance von Cloud-Diensten.

VM (Virtual Machine):

VM steht für Virtual Machine und bezeichnet eine virtuelle Instanz eines Computersystems.

Virtual Machine Scale Set:

Virtual Machine Scale Set (VMSS) ermöglicht die automatische Skalierung einer Gruppe von virtuellen Maschinen in Azure, um die Nachfrage zu erfüllen.

Virtual Network:

Ein Virtual Network (Virtuelles Netzwerk) ist eine logische Isolationseinheit in der Cloud, die es ermöglicht, Ressourcen wie virtuelle Maschinen, Container und Dienste innerhalb einer definierten Netzwerkumgebung zu verbinden und zu isolieren. Es ermöglicht die sichere Kommunikation zwischen diesen Ressourcen und kann auch Verbindung mit lokalen Netzwerken herstellen.

VPN Gateway:

Ein VPN-Gateway ermöglicht eine sichere Verbindung zwischen lokalen Netzwerken und Cloud-Netzwerken über das Internet.

Zone:

Eine Zone ist eine physische Region innerhalb eines Rechenzentrums, die über ihre eigenen Ressourcen verfügt.
